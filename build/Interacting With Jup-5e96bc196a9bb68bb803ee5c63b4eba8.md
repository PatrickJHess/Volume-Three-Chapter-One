# **🪐 Working with Jupyter Books**

Two types of documents are utilized across this and other chapters to deliver content and code:

### **📄 1\. Markdown Documents (.md)**

* **What it is:** Static text (like what you are currently reading).  
* **Code:** Contains no executable code.  
* **Downloads:** Available to save as a Markdown (.md) or PDF (.pdf) file.

### **🧑‍💻 2\. Notebook Documents (.ipynb)**

* **What it is:** HTML representations of interactive Jupyter Notebooks.  
* **Code:** Fully interactive. You can run code inside the HTML file or via Binder using Jupyter Lab.  
* **Downloads:** Available to save as a Notebook (.ipynb) or PDF (.pdf) file.

## **🚀 Notebook Launch Options**

For .ipynb files, you have three ways to interact with the code:

| Option | Description | Interactive Features | Keep in Mind |
| :---- | :---- | :---- | :---- |
| 📦 **Binder** | Launches remotely in a new tab. | Execute, add, modify, and download. | Changes are not saved after closing (ephemeral container). |
| ☁️ **Colab** | Fast access via Google's cloud. | Execute cells or save a copy. | Requires a Google login to run or copy. |
| ⚡ **Live Code** | Runs directly on the page via Binder. | Execute existing code. | Cannot add, modify, or save cells. |

## **🎛️ Additional Interactive Features**

* ⏯️ **Run Live Code:** Click the **start button** at the top of a notebook page to execute code directly inside the HTML file.  
* 🛸 **Quick Launch:** The first cell in every notebook includes direct links to open the environment in **Colab** or **Binder**.  
* 📋 **One-Click Copy:** Hover over the upper-right corner of any code cell or block to quickly copy the script to your clipboard.

## **🎯 Summary**

By combining narrative Markdown with interactive Notebooks, Jupyter Books seamlessly bridge the gap between financial theory and real-world applications.
